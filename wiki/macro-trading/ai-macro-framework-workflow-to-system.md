# AI 宏观投研系统：从流程到系统

> Sources: Dorian君 / 灰岩金融科技, 2026-04-26

## Overview

将投研流程升级为自动运转的研究引擎：10 条可直接用的 AI 投研 Prompt + 三层系统架构（记忆层/触发层/执行层）+ 五层宏观系统结构（数据输入→事件触发→流程→情景库→交易映射→风险控制→反馈）。核心观点：没有系统，AI 是噪音放大器；有系统，AI 是杠杆。

## 流程 vs 系统

| | 流程（手动模式） | 系统（自动运行） |
|--|----------------|----------------|
| 本质 | 每天从零开始 | 持续积累，触发驱动 |
| 依赖 | 靠意志力执行 | 靠机制自动运转 |
| 差距 | 会分析但赚不到钱 | 结构清晰，认知进化 |

系统 = 流程 + 三层：记忆层（Memory）、触发层（Trigger）、执行层（Execution）。

## 10条可直接用的AI投研Prompt

每天用，三个月后开始意识到大多数人根本不知道自己在看什么。

1. **宏观事件拆解**：请拆解【某个宏观事件】影响的关键变量，并按重要性排序。
2. **变量传导路径**：这些变量之间如何传导？请用因果链形式表达。
3. **市场结构影响**：这些变量变化，会如何影响利率、汇率、股市、商品？
4. **历史类似情景**：历史上是否出现过类似结构？对应市场表现如何？
5. **情景推演**：如果【变量A发生】但【变量B不跟随】，意味着什么？
6. **市场在交易什么**：当前市场最可能在交易的核心逻辑是什么？是否与表面新闻一致？
7. **交易表达**：如果上述逻辑成立，对应的交易路径是什么？（标的 + 方向 + 逻辑）
8. **反向推演**：如果这个逻辑是错的，最可能错在哪里？
9. **风险失效条件**：哪些变量变化，说明这个交易逻辑已经失效？
10. **压缩总结**：请用3句话总结当前结构、核心驱动、交易方向。

## AI宏观系统结构

```
数据输入层 → 事件触发层 → 宏观研究流程6步 → 情景库 → 交易映射层 → 风险控制层 → 反馈系统
```

### 数据输入层

**问题**：输入是随机的。**解法**：固定输入——利率（美债）、美元（DXY）、商品（油/金）、信用利差、宏观事件（央行/政策）。AI 压缩信息、提取变量、标准化输入。不是在看新闻，是在喂系统。

### 事件触发层

**问题**：没有触发机制，就是被市场拖着走。**解法**：定义什么叫"值得分析"——利率变动超过某阈值、汇率突破区间、商品异常波动、流动性变化。触发后才分析。

### 流程层

被触发后才有价值。AI 快速建模变量、生成因果链、推演情景。不再猜涨跌，运行结构模型。

### 情景库（护城河）

记录事件、变量、情景、结果，变成"认知数据库"。AI 检索历史结构、做对比、提炼规律。**决定你是否会进化**。

### 交易映射

分析很安全，交易要承担责任。必须强制情景 → 交易表达（标的 + 方向 + 逻辑）。否则只是"会分析但赚不到钱的人"。

### 风险系统（分水岭）

- **普通人**：会不会涨？
- **专业交易者**：我什么时候错？

定义失效条件、时间窗口、结构变化。AI 帮助找漏洞、推演反向路径。风险不是保护，是让你活下来。

### 反馈系统

每次之后问：哪里错了？哪个变量关键？哪个逻辑有效？然后写回系统。优化的不是交易，是**判断函数**——判断力本身。

## AI的正确用法

| 普通人 | 高手 |
|--------|------|
| 问答案 | 让AI参与过程 |

AI 擅长：扫描、推演、压缩、建模。AI 不是预测工具，是**认知外骨骼**。

## 每日最小执行框架

每天只做5件事：
1. 找一个触发事件
2. 跑6步流程
3. 写情景 + 交易
4. 定义风险
5. 复盘

**时间线**：30天开始看懂市场 → 90天形成判断力 → 一年拥有自己的系统。

**核心原则**：市场不奖励信息，奖励结构清晰的人。没有系统：AI是噪音放大器。有系统：AI是杠杆。

# 机器学习核心概念

> 来源: 用户笔记, 2026-04-27; Various blog posts

## Embedding 的作用

### 1. 降维

相比于 one-hot 编码，通过矩阵变化解决数据稀疏问题。

### 2. 分类

在矩阵变化映射中，切分稀疏特征、聚合稠密特征。

### 3. 可计算

每个特征都是一个向量，通过计算向量间距离（余弦）即可获得其关联度。

## 数据取对数的意义

1. **缩小资料的绝对数值**，方便计算
2. **对数值小的部分差异的敏感程度**比数值大的部分的差异敏感程度更高
3. 压缩了变量的尺度，资料更加平稳，不改变资料的性质和相关关系

## 监督 / 半监督 / 无监督学习

### 无监督学习（Unsupervised Learning）

没有给定事先标记过的训练范例，自动对输入的资料进行分类或分群。

主要运用：
- 聚类分析（Cluster Analysis）
- 关联规则（Association Rule）
- 维度缩减（Dimensionality Reduce）

典型算法：GAN、SOM、ART（自组织映射、适应性共振理论）

### 半监督 / 自监督学习（Self-Supervised Learning）

一个样本中的内容特别多，用一个样本做多个任务，等于把其他内容充分利用起来。

例如：遮挡图片的特定部分，用未遮挡部分预测遮挡部分。通过遮挡不同部分，用一个样本完成不同任务。Yann LeCun 将此方法称为自监督学习。

### 监督式学习（Supervised Learning）

由标记好的训练集学到或建立一个模式（函数），并依此模式推测新的实例。

- **回归**：输出是连续值
- **分类**：输出是预测标签

## 对比学习（Contrastive Learning）

一种自监督学习方法，通过比较不同输入之间的相似性和差异性来学习特征表示。

### 核心思想

从 ImageNet 中抽出猫、猫、狗、飞机四张图：
- 猫和猫的图片肯定是相似的
- 猫和狗不相似，但与飞机比起来，猫和狗更相似

模型不需要真的知道图片中代表的是什么，只需要知道哪些图片类似、哪些图片不同。

### Pretext Task（代理任务）

对比学习不需要标签（不需要知道图片是哪一类），但模型需要知道哪些图片类似才能训练。通过设计代理任务来人为指定：

- **Instance Discrimination**（实例判别）：从一堆图片中抽出任意一张做一次转换（如随机裁剪）得到新图片
  - 原图 Xi → 基准点（锚点）
  - 转换后 Xi1 → 正样本（从 Xi 变化得到，语义信息不变）
  - 数据集中其它所有图片 → 负样本

## 强化学习

一种试错方法，目标是让软件智能体在特定环境中采取回报最大化的行为。

主要技术：
- 动态规划（DP）
- 时间差分（TD）学习
- SARSA 算法
- Q 学习
- 深度强化学习（DQN）

应用：下棋类游戏、机器人控制、工作调度等。

## Instruct 的改进（ChatGPT 的贡献）

ChatGPT 的最大贡献在于：**基本实现了理想 LLM 的接口层，让 LLM 适配人的习惯命令表达方式**，而不是让人去适配 LLM。

### Prompting 方法演进

| 方法 | 说明 |
|------|------|
| **Prompting（早期）** | 人绞尽脑汁想出能 Work 的命令，尝试不同单词或句子反复测试 |
| **Zero Shot / Instruct** | 用自然语言给出任务描述（如"把这个句子从中文翻译成英文"） |
| **Few Shot / In Context Learning** | 给 LLM 几个示例作为范本，让 LLM 解决新问题 |

Instruct 是一种抽象的描述方式，In Context Learning 是一种例子示范法。

核心区别：Instruct 试图让 LLM 理解任务描述，而早期 Zero Shot Prompting 实际上是在拟合训练数据分布。

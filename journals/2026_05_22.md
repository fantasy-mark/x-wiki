## git手冊
	- ### 基本使用
	- #### 1.1 配置與初始化
	  collapsed:: true
		- ```
		  # 全局配置
		  git config --global user.name "Your Name"
		  git config --global user.email "your@email.com"
		  git config --global init.defaultBranch main
		  git config --global pull.rebase true
		  git config --global color.ui auto
		  
		  # 初始化倉庫
		  git init
		  # 或克隆遠程倉庫
		  git clone <url>
		  git clone -b <分支名> <url>
		  ```
	- #### 1.2 日常操作
		- | 命令                                | 作用         |
		  | --------------------------------- | ---------- |
		  | `git status`                      | 查看當前狀態     |
		  | `git add .`                       | 添加所有修改到暫存區 |
		  | `git add <文件>`                    | 添加指定文件     |
		  | `git commit -m "訊息"`              | 提交暫存區內容    |
		  | `git push`                        | 推送到遠程      |
		  | `git pull`                        | 拉取並合併遠程更新  |
		  | `git log --oneline`               | 簡潔提交歷史     |
		  | `git log --oneline --graph --all` | 圖形化分支歷史    |
	- #### 1.3 撤銷與回退
		- ```shell
		  # 撤銷工作區修改（未 add）
		  git restore <文件>
		  
		  # 撤銷暫存區修改（已 add 未 commit）
		  git restore --staged <文件>
		  
		  # 撤銷最後提交（保留修改）
		  git reset --soft HEAD~1
		  
		  # 撤銷最後提交（丟棄修改）
		  git reset --hard HEAD~1
		  
		  # 修改最後一次提交
		  git commit --amend -m "新訊息"
		  
		  # 查看所有操作記錄
		  git reflog
		  ```
	- #### 1.4 儲存（Stash）
		- ```shell
		  git stash push -m "描述"      # 儲存當前修改
		  git stash list                # 查看儲藏列表
		  git stash pop                 # 恢復並刪除最新儲藏
		  git stash apply stash@{0}     # 恢復指定儲藏
		  git stash drop stash@{0}      # 刪除指定儲藏
		  ```
	- ### 團隊合作：Git Flow 管理
	- #### 2.1 Git Flow 分支結構
		- ```shell
		  master（生產環境，隨時可部署）
		  │
		  └── develop（開發主線，集成最新成果）
		      │
		      ├── feature/user-login      → 合併回 develop
		      ├── feature/payment-system  → 合併回 develop
		      │
		      ├── release/v1.2.0          → 合併到 master + develop
		      │
		      └── hotfix/critical-bug     → 合併到 master + develop
		  ```
	- #### 2.2 各分支職責
		- | 命令                                      | 作用           |
		  | --------------------------------------- | ------------ |
		  | `git worktree add <路徑> <分支>`            | 基於現有分支創建工作目錄 |
		  | `git worktree add -b <新分支> <路徑> <基礎分支>` | 創建新分支並建立工作目錄 |
		  | `git worktree add <路徑> origin/<分支>`     | 基於遠程分支創建     |
		  | `git worktree list`                     | 列出所有工作目錄     |
		  | `git worktree remove <路徑>`              | 刪除工作目錄       |
		  | `git worktree remove -f <路徑>`           | 強制刪除         |
		  | `git worktree prune`                    | 清理失效記錄       |
		  | `git worktree lock <路徑>`                | 鎖定防止自動清理     |
		  | `git worktree move <舊> <新>`             | 移動工作目錄位置     |
	- #### 2.3 完整操作流程
		- ```shell
		  # ========== 同時處理主線 + 緊急修復 + Review ==========
		  
		  cd ~/projects/myapp
		  
		  # 1. 創建緊急修復工作目錄
		  git worktree add -b hotfix/payment-fix ../myapp-hotfix main
		  
		  # 2. 創建 Review 工作目錄
		  git worktree add ../myapp-review origin/pr/123
		  
		  # 3. 查看所有工作目錄
		  git worktree list
		  # /home/user/projects/myapp          a1b2c3d [main]
		  # /home/user/projects/myapp-hotfix   e4f5g6h [hotfix/payment-fix]
		  # /home/user/projects/myapp-review   i7j8k9l [(HEAD detached)]
		  
		  # ========== 三個目錄同時工作 ==========
		  
		  # 目錄 1：主線開發
		  cd ~/projects/myapp
		  # 編輯... → git add . → git commit -m "feat: ..."
		  
		  # 目錄 2：修復緊急 Bug
		  cd ~/projects/myapp-hotfix
		  # 修復... → git add . → git commit -m "fix: ..."
		  # → git push origin hotfix/payment-fix → 創建 PR 合併
		  
		  # 目錄 3：Review 同事程式碼
		  cd ~/projects/myapp-review
		  npm test
		  # 寫 Review 評論
		  
		  # ========== 清理 ==========
		  cd ~/projects/myapp
		  git worktree remove ../myapp-hotfix
		  git worktree remove ../myapp-review
		  git worktree prune
		  ```
	- #### 2.4 提交訊息規範（Conventional Commits）
		- ```shell
		  <type>(<scope>): <subject>
		  
		  類型：
		  feat:     新功能
		  fix:      修復 bug
		  docs:     文檔更新
		  style:    代碼格式調整
		  refactor: 重構
		  test:     添加/修改測試
		  chore:    維護任務
		  perf:     效能改進
		  ci:       CI/CD 變更
		  revert:   還原提交
		  
		  範例：
		  git commit -m "feat(auth): add OAuth2 login"
		  git commit -m "fix(api): handle null response"
		  ```
	- #### 2.5 團隊協作黃金法則
		- `master`/`main` 禁止直接推送，必須通過 PR/MR 合併
		- 至少 1 人審查通過才能合併
		- CI 測試失敗禁止合併
		- 每天合併，避免長期分支
		- 合併後立即刪除本地和遠程分支
	- ### Worktree 使用
	- #### 3.1 核心概念
		- ```shell
		  ┌─────────────────────────┐
		  │      共享 Git 資料庫     │
		  │      （只存一份）        │
		  │  ┌─────┐    ┌─────┐     │
		  │  │目錄A│    │目錄B │     │
		  │  │main │    │feat │     │
		  │  └─────┘    └─────┘     │
		  └─────────────────────────┘
		  ```
	- #### 3.2 常用命令
		- | 命令                                      | 作用           |
		  | --------------------------------------- | ------------ |
		  | `git worktree add <路徑> <分支>`            | 基於現有分支創建工作目錄 |
		  | `git worktree add -b <新分支> <路徑> <基礎分支>` | 創建新分支並建立工作目錄 |
		  | `git worktree add <路徑> origin/<分支>`     | 基於遠程分支創建     |
		  | `git worktree list`                     | 列出所有工作目錄     |
		  | `git worktree remove <路徑>`              | 刪除工作目錄       |
		  | `git worktree remove -f <路徑>`           | 強制刪除         |
		  | `git worktree prune`                    | 清理失效記錄       |
		  | `git worktree lock <路徑>`                | 鎖定防止自動清理     |
		  | `git worktree move <舊> <新>`             | 移動工作目錄位置     |
	- #### 3.3 完整實戰案例
		- ```shell
		  # ========== 同時處理主線 + 緊急修復 + Review ==========
		  
		  cd ~/projects/myapp
		  
		  # 1. 創建緊急修復工作目錄
		  git worktree add -b hotfix/payment-fix ../myapp-hotfix main
		  
		  # 2. 創建 Review 工作目錄
		  git worktree add ../myapp-review origin/pr/123
		  
		  # 3. 查看所有工作目錄
		  git worktree list
		  # /home/user/projects/myapp          a1b2c3d [main]
		  # /home/user/projects/myapp-hotfix   e4f5g6h [hotfix/payment-fix]
		  # /home/user/projects/myapp-review   i7j8k9l [(HEAD detached)]
		  
		  # ========== 三個目錄同時工作 ==========
		  
		  # 目錄 1：主線開發
		  cd ~/projects/myapp
		  # 編輯... → git add . → git commit -m "feat: ..."
		  
		  # 目錄 2：修復緊急 Bug
		  cd ~/projects/myapp-hotfix
		  # 修復... → git add . → git commit -m "fix: ..."
		  # → git push origin hotfix/payment-fix → 創建 PR 合併
		  
		  # 目錄 3：Review 同事程式碼
		  cd ~/projects/myapp-review
		  npm test
		  # 寫 Review 評論
		  
		  # ========== 清理 ==========
		  cd ~/projects/myapp
		  git worktree remove ../myapp-hotfix
		  git worktree remove ../myapp-review
		  git worktree prune
		  ```
	- #### 3.4 裸倉庫管理模式（推薦長期維護）
		- ```shell
		  # 創建裸倉庫作為管理中心
		  git clone --bare https://github.com/user/project.git project.git
		  cd project.git
		  
		  # 添加各個工作目錄
		  git worktree add ../project-main main
		  git worktree add ../project-dev develop
		  git worktree add ../project-release release/1.2.0
		  
		  # 目錄結構：
		  # project.git/        ← 管理中心（無工作檔案）
		  # project-main/       ← main 分支
		  # project-dev/        ← develop 分支
		  # project-release/    ← release 分支
		  ```
	- #### 3.5 注意事項
		- | 限制                       | 解決方案                    |
		  | ------------------------ | ----------------------- |
		  | 一個分支只能被一個工作目錄檢出          | 建立臨時分支                  |
		  | `node_modules` 等未追蹤檔案不共享 | 每個目錄獨立安裝                |
		  | 工作目錄不能嵌套                 | 所有目錄放在同一層級              |
		  | 手動刪除目錄後需清理               | 執行 `git worktree prune` |
- ## docker、docker-compose手冊
	- ### Docker 基础
	- #### 1. 镜像操作
		- ```shell
		  # 搜索镜像
		  docker search nginx
		  
		  # 拉取镜像
		  docker pull nginx:latest
		  
		  # 查看本地镜像
		  docker images
		  
		  # 删除镜像
		  docker rmi nginx:latest
		  
		  # 构建镜像（当前目录需有 Dockerfile）
		  docker build -t myapp:1.0 .
		  ```
	- #### 2. 容器操作
		- ```shell
		  # 运行容器
		  docker run -d -p 80:80 --name mynginx nginx
		  
		  # 常用参数说明：
		  # -d        后台运行
		  # -p        端口映射（主机端口:容器端口）
		  # --name    指定容器名称
		  # -v        挂载卷（主机路径:容器路径）
		  # -e        设置环境变量
		  # --restart 自动重启策略
		  
		  # 查看运行中的容器
		  docker ps
		  
		  # 查看所有容器（包括停止的）
		  docker ps -a
		  
		  # 停止/启动/重启容器
		  docker stop mynginx
		  docker start mynginx
		  docker restart mynginx
		  
		  # 进入容器内部
		  docker exec -it mynginx /bin/bash
		  
		  # 查看容器日志
		  docker logs -f mynginx
		  
		  # 删除容器
		  docker rm mynginx
		  
		  # 强制删除运行中的容器
		  docker rm -f mynginx
		  ```
		- 容器状态流转图
		  ```
		  docker create    docker start      docker stop
		      ───►  Created  ───►  Up/Running  ───►  Exited
		             ▲                               │
		             └──────── docker start ─────────┘
		  ```
	- #### 3. 数据卷
		- ```shell
		  # 创建卷
		  docker volume create mydata
		  
		  # 查看卷
		  docker volume ls
		  
		  # 挂载卷运行
		  docker run -d -v mydata:/data nginx
		  
		  # 绑定挂载主机目录
		  docker run -d -v /host/path:/container/path nginx
		  ```
- ### Docker Compose 基础
	- #### 1. 基本配置文件示例
	  collapsed:: true
		- ```yaml
		  # docker-compose.yml
		  version: '3.8'
		  
		  services:
		    web:
		      image: nginx:latest
		      container_name: my-web
		      ports:
		        - "80:80"
		      volumes:
		        - ./html:/usr/share/nginx/html
		      networks:
		        - mynet
		      depends_on:
		        - db
		  
		    db:
		      image: mysql:8.0
		      container_name: my-db
		      environment:
		        MYSQL_ROOT_PASSWORD: secret
		        MYSQL_DATABASE: mydb
		      volumes:
		        - db_data:/var/lib/mysql
		      networks:
		        - mynet
		  
		    redis:
		      image: redis:alpine
		      container_name: my-redis
		      ports:
		        - "6379:6379"
		      networks:
		        - mynet
		  
		  volumes:
		    db_data:
		  
		  networks:
		    mynet:
		      driver: bridge
		  ```
	- #### 2. 常用命令
		- ```shell
		  # 启动所有服务（后台运行）
		  docker-compose up -d
		  
		  # 启动并重新构建镜像
		  docker-compose up -d --build
		  
		  # 查看服务状态
		  docker-compose ps
		  
		  # 查看服务日志
		  docker-compose logs -f
		  
		  # 停止服务
		  docker-compose stop
		  
		  # 停止并删除容器、网络
		  docker-compose down
		  
		  # 停止并删除容器、网络、卷（慎用）
		  docker-compose down -v
		  
		  # 重启服务
		  docker-compose restart
		  
		  # 进入某个服务容器
		  docker-compose exec web /bin/bash
		  
		  # 构建或重建服务
		  docker-compose build
		  
		  # 拉取最新镜像
		  docker-compose pull
		  
		  # 指定文件运行
		  docker-compose -f production.yml up -d
		  
		  # 僅啟動某個服務
		  docker-compose up server_name
		  ```
	- #### 3. 进阶配置示例`Dockerfile` 只能访问 `context` 目录及其子目录下的文件
		- ```yaml
		  # 包含健康检查、资源限制、环境变量文件
		  version: '3.8'
		  
		  services:
		    app:
		      build:
		        # `Dockerfile` 只能访问 `context` 目录及其子目录下的文件
		        context: ./app
		        dockerfile: Dockerfile
		      env_file:
		        - .env
		      deploy:
		        resources:
		          limits:
		            cpus: '1.0'
		            memory: 512M
		      healthcheck:
		        test: ["CMD", "curl", "-f", "http://localhost:8080/health"]
		        interval: 30s
		        timeout: 10s
		        retries: 3
		        start_period: 40s
		      restart: unless-stopped
		  ```
